# MFA Pages

PlusAuth Widget supports multiple MFA views by default:
```ts
[
    "sms", //SMS verification
    "email", //Email verification
    "otp", //OTP verification
    "push", //Push Notification
    "sc", //Smart Card Verification
    "webauthn", //Security key or device
    "fv" //FingerVein
]
```


When the widget mode is plain `'mfa'` as such: 

```javascript
const auth = new PlusAuthWidget(document.body, {/* configurations here */}, window.PlusAuth)
auth.view.mode = 'mfa'
```

The widget will land on the MFA Challenge page. This page will direct the user to the selected view. 

If you wish the user to directly land on a specific MFA view, you need to set the mode to the specific view like this: 

```javascript
const auth = new PlusAuthWidget(document.body, {/* configurations here */}, window.PlusAuth)
auth.view.mode = 'mfa-sms'
```

Views like [OTP](?path=/story/pages-mfa--otp-challenge) and [Push](?path=/story/pages-mfa--push-challenge) have prompts for certain uses. 
The mechanism to render these prompts is linked with `context.details`:

```javascript

context: {
    //...
    details: {
        /**
         * For QR Setup
        */
        dataUrl: '<< QR Code URL >>',

        /**
         * For Manual Setup
        */
        dataUrl: '<< QR Code URL >>',
        secret: '<< Secret String >>'
    }
    //...
}
```

If nothing is provided under `context.details`, the view will be rendered in standard mode.
