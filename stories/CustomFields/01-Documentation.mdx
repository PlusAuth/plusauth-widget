# Custom Fields

Use `modeOptions.[pageIdentifier].fields` to add, remove, reorder, or override fields for a specific page.

Common page keys:

`login`, `signup`, `recovery`, `resetPassword`, `fillMissing`, `verifyEmail`, `passwordChallenge`, `emailMfa`, `smsMfa`, `otpMfa`, `pushMfa`, `webauthnMfa`, `fvMfa`, `passwordlessEmail`, `passwordlessSms`, `passwordlessOtp`, `passwordlessPush`, `passwordlessWebauthn`, `accountLinking`

## Basic Usage

```javascript
new PlusAuthWidget('#widget', {
  modeOptions: {
    login: {
      fields: {
        'custom-text-field': {
          type: 'text',
          label: 'Username'
        }
      }
    }
  }
})
```

To remove a default field, set it to a falsy value:

```javascript
{
  modeOptions: {
    login: {
      fields: {
        email: undefined
      }
    }
  }
}
```

## Action Helpers

`preAction` and `postAction` can run before and after a page action.

```javascript
new PlusAuthWidget('#widget', {
  modeOptions: {
    resetPassword: {
      preAction(values) {
        if (values.password.includes('something')) {
          throw { error: 'my_custom_error' }
        }
      },
      postAction() {
        window.location.replace('https://myapplication.url/after-reset-password')
      }
    }
  }
})
```

Use `responseErrorHandler` for custom error handling:

```javascript
new PlusAuthWidget('#widget', {
  modeOptions: {
    resetPassword: {
      responseErrorHandler(error, formRef) {
        if (error && error.error === 'my_custom_error') {
          formRef.toggleAlert('My custom text')
        }
      }
    }
  }
})
```

## Advanced Field Options

`required`: defaults to `true`. Set `required: false` to make a field optional.

`visible`: set `false` or `'hidden'` to hide a field while still keeping value behavior.

`ignored`: when `true`, skips validation and omits the field from submitted payload.

`order`: controls rendering order.

`attrs`: passes HTML/input props through to the rendered widget field.

`slots`: allows injecting component content into input slots.

```javascript
{
  modeOptions: {
    login: {
      fields: {
        username: {
          type: 'text',
          label: 'Username',
          required: false,
          order: 2,
          slots: {
            append: {
              element: 'PSpinner',
              props: {
                color: 'primary',
                indeterminate: true
              }
            }
          }
        }
      }
    }
  }
}
```

## Input Attributes

Use `attrs` for practical field-level behavior such as autocomplete, disabling a field, or hiding helper messages.

```javascript
{
  modeOptions: {
    login: {
      fields: {
        email: {
          type: 'text',
          label: 'Email',
          attrs: {
            autocomplete: 'username',
            hideMessages: true,
            disabled: false
          }
        }
      }
    }
  }
}
```

Common useful values:

- `autocomplete`
- `disabled`
- `readonly`
- `inputmode`
- `pattern`
- `hideMessages`

## Custom Validation

Use `validator` for sync or async validation. Return `true` for pass or return a string error key/message.

```javascript
{
  fields: {
    password: {
      type: 'password',
      validator(fields, value) {
        if (!value || value.length < 8) {
          return 'Password must be at least 8 characters'
        }
        return true
      }
    }
  }
}
```
